3 User Interface

Although this application runs in a web browser, it behaves much like a desktop application. For example, the paradigms of drag&drop and context menus are used throughout the application.

3.1 Login


At the login screen, the login and password credentials are entered. They are checked against the system database. If successful, The user is automatically forwarded to the workbench. Otherwise, an error message is displayed.

3.2 Workbench


The workbench consists of several areas, called views. All views except the main model view can be collapsed by the » buttons in the title bar.

3.2.1 New Objects

The new objects view is located in the upper left part of the workbench. It lists all model objects the user may add to a diagram. For each object, an icon, the title and a description is listed.

Task

Action

Description

Add new model object to diagram

Drag new object from new objects view, drop on diagram view

Adds a new model object of the selected type to the model at the dropped location. The model object is automatically added to the package containing the model the user dropped the new object on.

3.2.2 Model Tree

The model tree view is the first tab in the lower left part of the workbench. It contains all models of this application. It shows the logical position of each object in the application.

Task

Action

Description

Create new model in application

Right-click on empty space in model tree, select “Create model” in context menu

Creates a new model in the application. The default name is assigned to the new model.

3.2.2.1 Models

Models are the root elements in the model tree.

Task

Action

Description

View model properties

Single-click on the model

All properties of a model can be reviewed in the properties view.

Change label of a model

Single-click on the model, change name in properties view, leave name field

If you change the model name in properties view, the model tree is automatically updated.

Add package to model

Right-click on the model, select “Add package” from context menu

Adds a package below the selected model. The default name is assigned to the new package.

Delete model

Right-click on the model, select “Delete from application” from context menu

Deletes the model and all contained packages, diagrams and model objects from the application.

List all model objects of specific types contained in this model

Right-click on the model, select “Select as grid scope” from the context menu

For some model object types, lists of all model objects of this type are available. This action selects a model (and all directly and indirectly contained packages) as scope for the lists.

The diagram view takes the center space of the workbench. It shows the diagrams, containing of model objects and their connections.

3.2.6 Properties

The properties view is located at the right part of the workbench. It shows the properties of a selected object. Each object has different properties.

3.2.6.1 Text Input Fields

Text input fields allow a single line of text. Changes are saved to the model when leaving the input field.

3.2.6.2 Dropdown Lists

Dropdown lists allow selection of a list of pre-defined values. Changes are saved to the model when leaving the input field.

3.2.6.3 Rich Text Edit Boxes

Richt text edit boxes allow multiple lines or paragraphs of formatted text. The formats bold, italic, underline, ordered list and unordered list are available. Changes are saved to the model when a different object's properties are displayed.

3.2.6.4 Read-Only Fields

Read-only fields are displayed with a grey background. They cannot be edited.
